2. 가급적이면 리턴값을 저장해놨다가 마지막에 한번만 쓰기.

3. JVM의 메모리 구조
- 3가지 주요영역(Method area, Call stack, Heap)
- Method area : 클래스 데이터 - 클래스 변수
- Call stack : 호출된 메서드들 위한 메모리 할당 - 지역 변수
- Heap : 인스턴스가 생성되는 공간 - 인스턴스 변수

4. 기본형(Primitive type) 매개변수와 참조형(Reference type) 매개변수
- 자바에서는 메서드 호출할 때 매개변수로 지정한 값을 메서드의 매개변수에 복사해서 넘긴다.
- 기본형은 Read Only, 참조형은 Read & Write

5.  언제 인스턴스메서드/클래스메서드로 정의할 것인가?
메서드 중 인스턴스 변수/메서드 사용하지 않는 메서드는 static 선언하는 게 좋다.
클래스메서드에서는 인스턴스변수/메서드 호출할 수 없다. 인스턴스가 생겼는지 안 생겼는지 모르기 때문이다.

6. Card c = new Card(); 를 실행하면 내부적으로 어떤 일이 일어나는가?
1) 연산자 new에 의해서 Heap에 Card 클래스의 인스턴스가 생긴다.
2) Constructor Card()가 호출된다.
3) 생성된 Card 인스턴스 주소가 반환되어 참조변수 c에 저장된다.
